# Метод Грефе для вещественных и комплексных корней

## Файлы

### Основные реализации
- **Graeffe_real_roots.h** — базовая реализация метода Греффе для вычисления модулей корней вещественных многочленов в логарифмической форме.
- **Graeffe_complex_roots.h** — модифицированный метод Греффе для определения комплексных корней (вещественная и мнимая части).
- **Graeffe_Jurchse.h** — реализация метода Греффе с уточняющей процедурой Юркша для повышения точности сближенных и кратных корней.
- **Graeffe_SanJuan.h** — реализация метода Греффе с нормализацией коэффициентов по схеме Сан-Хуана для улучшенной численной устойчивости.

### Тестовые файлы
- **graeffe_real_roots_test.cpp** — тестирование базового метода на вещественных корнях.
- **graeffe_complex_roots_test.cpp** — тестирование метода комплексных корней.
- **graeffe_jurchse_test.cpp** — тестирование метода с уточнением Юркша.
- **graeffe_sanjuan_test.cpp** — тестирование метода с нормализацией Сан-Хуана.

## Назначение
Разработанный программный модуль представляет набор реализаций метода Греффе для вычисления корней алгебраических многочленов с действительными коэффициентами. Основой всех реализаций является алгоритм Греффе, выполненный в устойчивой логарифмической форме.

Программа позволяет:
1. находить модули корней многочлена;
2. определять вещественные и комплексные корни;
3. определять кратности корней по совпадающим модулям;
4. работать с коэффициентами различных числовых типов (float, double, long double и произвольной точности).

---

## Алгоритмические особенности базового метода (Graeffe_real_roots.h)

1. **Использование логарифмов**  
   Все операции выполняются над логарифмами коэффициентов для предотвращения переполнения при многократном возведении в квадрат.

2. **Функция безопасного логарифма**  
   Для обработки нулевых коэффициентов используется `safe_log(x)`, возвращающая `-inf` при `x = 0`.

3. **Критерий завершения**  
   Итерации прекращаются, когда относительное изменение коэффициентов меньше `epsilon` или достигнут `maxIter`.

4. **Обработка особых случаев**  
   - Константный многочлен — результат пустой.  
   - Все коэффициенты равны нулю — результат пустой.  
   - Наличие промежуточных нулевых коэффициентов обрабатывается без ошибок.

---

## Алгоритмические особенности метода комплексных корней (Graeffe_complex_roots.h)

1. **Обработка различных случаев**  
   Вещественные корни, комплексные корни, кратные корни, нулевые корни.

2. **Численная устойчивость**  
   Использование логарифмической арифметики, адаптивные пороги для разных масштабов коэффициентов, множественная валидация кандидатов.

3. **Оптимизации**  
   Предвычисление производных полинома, кэширование промежуточных результатов, дедупликация найденных корней.

4. **Работа на основе метода Греффе**  
   В `Graeffe_real_roots.h` вычисляются модули всех корней (в т.ч. комплексных), и на их основе находятся действительная и мнимая части комплексного числа.

---

## Алгоритмические особенности метода Юркша (Graeffe_Jurchse.h)

1. **Уточняющая процедура Юркша**  
   После стандартных итераций Греффе применяется итерационный метод для уточнения корней, что позволяет достичь более высокой точности для сближенных и кратных корней.

2. **Локализация корней**  
   Используются кольцевые области для разделения корней с близкими модулями.

3. **Комбинированный подход**  
   Метод сочетает глобальную сходимость метода Греффе с локальной квадратичной сходимостью уточняющей процедуры.

---

## Алгоритмические особенности метода Сан-Хуана (Graeffe_SanJuan.h)

1. **Нормализация коэффициентов**  
   На каждом шаге итерации производится масштабирование коэффициентов для поддержания их в рабочем диапазоне значений.

2. **Динамическое масштабирование**  
   Коэффициенты приводятся к единичному масштабу по максимальному значению, что предотвращает переполнение и потерю значимости.

3. **Повышенная устойчивость**  
   Схема Сан-Хуан позволяет обрабатывать многочлены более высокой степени по сравнению с базовой реализацией.

---

## Результаты тестирования (базовая реализация)

Тестирование проведено на пяти группах полиномов различной степени и структуры. Все тесты выполнены с параметрами `epsilon = 1e-10`, `maxIter = 100`.

Реализация метода Греффе показала устойчивую и предсказуемую работу при вычислении модулей корней многочленов различной степени и структуры. Алгоритм корректно определяет модули корней для большинства тестовых полиномов и демонстрирует высокую численную устойчивость благодаря использованию логарифмической формы вычислений.

При полиномах малой и средней степени метод обеспечивает высокую точность и быструю сходимость. Для полиномов большой степени (более 20) результаты сохраняют корректность, хотя наблюдается частичное сглаживание различий между малыми модулями. В случаях наличия кратных или сильно сближенных корней точность определения отдельных модулей снижается, что соответствует известным ограничениям классического метода Греффе.

---

## Результаты тестирования (модифицированный метод для комплексных корней)

Тестирование модифицированного метода Грефе проведено на трёх типах данных (double, long double, float_precision) с параметрами `epsilon = 1e-10`, `maxIter = 100`. Оценка качества выполнялась по критерию максимальной относительной ошибки среди всех найденных модулей корней.

---

## Анализ результатов

Модифицированный метод Хосмана в целом подтвердил свою работоспособность и устойчивость в стандартных случаях, однако не прошёл все тесты с одинаковым качеством.

### Успешные сценарии
- Полиномы средней степени (до 30) с вещественными и комплексными корнями.
- Кластеризованные корни при степени до 20.
- Высокая точность на типах с увеличенной разрядностью (float_precision).

### Проблемные сценарии
- Полиномы высокой степени (>= 50) с вещественными корнями — накопление ошибок округления приводит к расходимости.
- Кратные корни (степени 10–20) — снижение точности из-за совпадения модулей.
- Типы double/long double демонстрируют меньшую устойчивость при высокой степени по сравнению с арифметикой произвольной точности.

### Выводы
Метод обеспечивает корректные результаты для простых и умеренно сложных многочленов, но демонстрирует снижение точности при высокой степени и кратных корнях. Дальнейшее развитие требует доработки алгоритмов нормализации коэффициентов и уточняющих процедур для повышения точности определения комплексных и близких по модулю корней.

Переход на арифметику произвольной точности (float_precision) значительно расширяет область применимости метода, позволяя успешно обрабатывать полиномы степени до 80 с приемлемой точностью.

## Сравнительная характеристика реализаций

| Реализация | Преимущества | Ограничения |
|------------|--------------|-------------|
| **Graeffe_real_roots.h** | Простота, логарифмическая устойчивость | Только модули корней, чувствительность к кратным корням |
| **Graeffe_complex_roots.h** | Определение комплексных корней | Снижение точности при высокой степени |
| **Graeffe_Jurchse.h** | Уточнение сближенных корней, квадратичная сходимость | Дополнительные вычислительные затраты |
| **Graeffe_SanJuan.h** | Нормализация коэффициентов, устойчивость для высоких степеней | Требует настройки параметров масштабирования |

## Рекомендации по выбору реализации

1. **Для простых многочленов малой и средней степени** — `Graeffe_real_roots.h` (достаточно модулей) или `Graeffe_complex_roots.h` (при необходимости комплексных корней).

2. **Для многочленов с близкими по модулю или кратными корнями** — `Graeffe_Jurchse.h` с уточняющей процедурой.

3. **Для многочленов высокой степени (более 30)** — `Graeffe_SanJuan.h` с нормализацией коэффициентов или использование арифметики произвольной точности.

4. **Для максимальной точности** — комбинация `Graeffe_SanJuan.h` с типом `float_precision`.
