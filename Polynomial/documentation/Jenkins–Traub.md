# Метод Дженкинса–Трауба (Jenkins–Traub) для вычисления корней многочленов

## Файл
- **JenkinsTraub.h** — реализация метода Дженкинса–Трауба (CPOLY) для нахождения всех корней алгебраических многочленов с действительными коэффициентами.

## Назначение
Метод Дженкинса–Трауба, известный как алгоритм CPOLY, является одним из наиболее надёжных и широко используемых алгоритмов для вычисления всех корней многочленов с действительными коэффициентами. Алгоритм был разработан М. А. Дженкинсом и Дж. Ф. Траубом в 1970 году и считается эталонным для этой задачи.

Программа позволяет:
1. находить все корни многочлена (вещественные и комплексные);
2. определять кратные корни;
3. работать с коэффициентами различных числовых типов (float, double, long double и произвольной точности).

---

## Алгоритмические особенности метода Дженкинса–Трауба

1. **Трёхэтапная структура**  
   Алгоритм состоит из трёх основных этапов:
   - **Этап 1 (No-shift)** — итерации без сдвига для улучшения разделения корней.
   - **Этап 2 (Fixed-shift)** — итерации с фиксированным сдвигом для уточнения очередного корня.
   - **Этап 3 (Variable-shift)** — итерации с переменным сдвигом для окончательного уточнения корня.

2. **Последовательное нахождение корней**  
   Корни находятся по одному, начиная с наименьшего по модулю. После нахождения каждого корня выполняется дефляция (понижение степени) многочлена.

3. **Обработка комплексных корней**  
   Алгоритм естественным образом работает с комплексными корнями, используя комплексную арифметику.

4. **Численная устойчивость**  
   - Использование масштабирования коэффициентов для предотвращения переполнения.
   - Применение адаптивных порогов сходимости.
   - Специальная обработка кратных корней.

5. **Обработка особых случаев**  
   - Константный многочлен — результат пустой.
   - Нулевые корни обрабатываются корректно.
   - Кратные корни определяются по сходимости.

## Выводы

### Ключевые результаты

1. **Метод Дженкинса–Трауба демонстрирует высокую точность на малых и средних степенях** — для многочленов до степени 20 невязка составляет 10⁻¹⁴ – 10⁻¹⁵ для типов double и long double, что является одним из лучших показателей среди рассмотренных методов.

2. **100% успешность на арифметике произвольной точности (float_precision)** — метод успешно прошёл все 8 тестов для типа float_precision, что подтверждает его устойчивость при использовании повышенной разрядности.

3. **Критическое ограничение для double/long double** — при степени 30 происходит резкая потеря точности (невязка достигает 1.33e+01), а при степени 80–100 возникают численные переполнения (inf). Это связано с ограниченной разрядностью стандартных типов с плавающей точкой.

4. **Устойчивость к кластеризации и кратным корням** — метод уверенно работает с близко расположенными и кратными корнями для степеней до 20.

5. **Эффективность для комплексных корней** — метод корректно находит комплексные корни, включая пары комплексно-сопряжённых корней, для степеней до 30 на double/long double.

## Особенности реализации

1. **Трёхэтапный алгоритм**  
   Реализация следует классической структуре CPOLY с этапами без сдвига, с фиксированным сдвигом и с переменным сдвигом.

2. **Масштабирование коэффициентов**  
   Перед началом вычислений коэффициенты многочлена масштабируются для улучшения численной устойчивости.

3. **Обработка вещественных корней**  
   Вещественные корни определяются с высокой точностью и могут быть извлечены без комплексной арифметики.

4. **Дефляция**  
   После нахождения каждого корня выполняется дефляция многочлена, что позволяет последовательно обрабатывать все корни.

## Заключение

Метод Дженкинса–Трауба (`JenkinsTraub.h`) является одним из наиболее надёжных алгоритмов для вычисления корней многочленов. В данной реализации метод демонстрирует:

- **Высокую точность** для многочленов степени до 20 на стандартных типах данных (double, long double);
- **Идеальную надёжность** на арифметике произвольной точности (float_precision);
- **Устойчивость** к кластеризованным и кратным корням.

Основным ограничением метода является чувствительность к разрядности арифметики: для многочленов степени выше 20 на double/long double наблюдается катастрофическая потеря точности. При использовании арифметики произвольной точности это ограничение снимается, что делает метод применимым для широкого круга задач.

Для достижения максимальной надёжности рекомендуется использовать метод Дженкинса–Трауба в сочетании с типом float_precision, особенно для многочленов высокой степени.
